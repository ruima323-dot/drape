# Requirements Document

## Introduction

Drape is a style discovery web application designed for users in their 20s and 30s who have not yet established a personal aesthetic. Unlike existing styling apps that assume a developed style, Drape helps users discover and evolve their style over time through AI-powered outfit generation, targeted accessory pairing, and longitudinal outfit tracking. The MVP focuses on wardrobe text input, outfit generation across occasion contexts, targeted accessory compositing, outfit saving, and an Outfit Journey timeline. Photo upload, CV-based wardrobe recognition, proactive daily suggestions, and affiliate buy links are deferred to v2.

## Glossary

- **Drape_App**: The Drape style discovery web application
- **User**: A person who has registered and authenticated with Drape_App
- **Wardrobe**: The collection of clothing items a User has described or uploaded to Drape_App
- **Wardrobe_Item**: A single clothing item stored as structured data containing item identifier, type, color, material, fit, and associated occasions
- **Accessory**: A fashion accessory item (e.g., earrings, necklace, bracelet, watch, hat, scarf) stored with type, color, material, label, and emoji representation
- **Occasion_Context**: One of three predefined styling contexts: Work, Casual, or Night Out
- **Outfit**: A combination of Wardrobe_Items and Accessories generated by Drape_App for a specific Occasion_Context, rendered as a visual image on an Avatar
- **Avatar**: A consistent visual representation of the User used to display generated Outfits, configured with approximate body type and skin tone
- **Accessory_Layer**: The composited image layer containing Accessories overlaid on the Avatar, which can be updated independently of the base Outfit image
- **Accessory_Shelf**: A persistent UI panel displaying the User's saved Accessories with toggle controls for adding or removing each Accessory from the current Outfit
- **Accessory_Prompt**: A natural language text input where the User describes an Accessory to add to the current Outfit (e.g., "add gold hoop earrings")
- **Outfit_Journey**: A chronological timeline view of all saved Outfits grouped by month
- **Saved_Outfit**: An Outfit that the User has explicitly saved, including an optional personal note, timestamp, and all associated metadata
- **Personal_Note**: An optional text annotation (maximum 280 characters) attached to a Saved_Outfit describing where the User wore it or how it felt
- **Wardrobe_Parser**: The component of Drape_App responsible for parsing plain text wardrobe descriptions into structured Wardrobe_Item data
- **Image_Generator**: The AI image generation service used by Drape_App to produce Outfit visuals (e.g., Stable Diffusion via Replicate API)
- **Accessory_Compositor**: The component responsible for updating only the Accessory_Layer of an Outfit image without regenerating the full Outfit
- **Recommendation_Engine**: The component that suggests Accessories based on the current Outfit, Wardrobe color palette, and historical Accessory usage
- **Style_Profile**: A data structure capturing the User's aesthetic baseline derived from the style questionnaire and accumulated Outfit history

## Requirements

### Requirement 1: Plain Text Wardrobe Entry

**User Story:** As a new user, I want to describe my wardrobe in plain text so I don't have to photograph every item.

#### Acceptance Criteria

1. WHEN a User submits a plain text wardrobe description, THE Wardrobe_Parser SHALL parse the text and create one or more Wardrobe_Items with structured fields: type, color, material, fit, and occasions.
2. WHEN a User submits a plain text wardrobe description containing multiple items, THE Wardrobe_Parser SHALL identify and separate each distinct clothing item into an individual Wardrobe_Item.
3. IF the Wardrobe_Parser cannot identify a valid clothing item from the submitted text, THEN THE Drape_App SHALL display an error message indicating which portion of the text could not be parsed.
4. WHEN a Wardrobe_Item is successfully parsed, THE Drape_App SHALL store the Wardrobe_Item in the database associated with the User's account.
5. FOR ALL valid plain text wardrobe descriptions, parsing then formatting then parsing the description SHALL produce equivalent Wardrobe_Items (round-trip property).

### Requirement 2: Wardrobe Catalogue Display

**User Story:** As a new user, I want to see my wardrobe items catalogued and displayed so I know the app understood me.

#### Acceptance Criteria

1. WHEN a User navigates to the My Wardrobe screen, THE Drape_App SHALL display all of the User's Wardrobe_Items in a browsable grid layout.
2. THE Drape_App SHALL display each Wardrobe_Item in the grid with its type, color, and material visible.
3. WHEN a User adds a new Wardrobe_Item, THE Drape_App SHALL update the wardrobe grid to include the new item without requiring a page reload.
4. WHEN a User has zero Wardrobe_Items, THE Drape_App SHALL display an empty state with a prompt to add wardrobe items via text input.

### Requirement 3: Style Questionnaire

**User Story:** As a new user, I want to answer a short style questionnaire so the app can learn my aesthetic baseline.

#### Acceptance Criteria

1. WHEN a new User completes the onboarding flow, THE Drape_App SHALL present a style questionnaire to establish the User's Style_Profile.
2. WHEN a User completes the style questionnaire, THE Drape_App SHALL store the responses as the User's Style_Profile.
3. THE Drape_App SHALL use the Style_Profile to influence Outfit generation and Accessory recommendations for the User.

### Requirement 4: Outfit Generation from Wardrobe and Context

**User Story:** As a user, I want to select an occasion context and see an AI-generated outfit visual so I can understand how my clothes look together.

#### Acceptance Criteria

1. WHEN a User selects an Occasion_Context (Work, Casual, or Night Out), THE Image_Generator SHALL generate a visual Outfit image using the User's Wardrobe_Items appropriate for the selected Occasion_Context.
2. THE Image_Generator SHALL render the Outfit on the User's Avatar, maintaining a consistent Avatar appearance across all Occasion_Contexts.
3. WHEN a User switches from one Occasion_Context to another, THE Image_Generator SHALL generate a new Outfit using the same Wardrobe without requiring the User to re-select wardrobe items.
4. WHEN an Occasion_Context is active, THE Drape_App SHALL display the Occasion_Context label as an overlay on the Avatar card.
5. WHEN a User switches Occasion_Context, THE Drape_App SHALL shift the background and color palette of the Avatar card to visually signal the context change.

### Requirement 5: Avatar Personalization

**User Story:** As a user, I want the avatar to reflect my approximate body type and skin tone so the outfit feels relevant to me.

#### Acceptance Criteria

1. THE Drape_App SHALL allow the User to configure the Avatar with approximate body type and skin tone settings.
2. WHEN a User updates Avatar configuration settings, THE Drape_App SHALL persist the updated Avatar configuration and apply the changes to all subsequent Outfit generations.
3. THE Image_Generator SHALL use the User's Avatar configuration to render Outfits on an Avatar that reflects the configured body type and skin tone.

### Requirement 6: Targeted Accessory Editing via Natural Language

**User Story:** As a user, I want to type "add gold hoop earrings" and see just that accessory appear on the avatar without the rest of the outfit changing so I can experiment quickly.

#### Acceptance Criteria

1. WHEN a User submits an Accessory_Prompt describing an accessory, THE Wardrobe_Parser SHALL parse the prompt to identify the accessory type, material, and color.
2. WHEN an Accessory is parsed from an Accessory_Prompt, THE Accessory_Compositor SHALL update only the Accessory_Layer of the current Outfit image to include the new Accessory.
3. WHEN the Accessory_Compositor updates the Accessory_Layer, THE Accessory_Compositor SHALL preserve the base Outfit image and Avatar without modification.
4. THE Drape_App SHALL display each active Accessory as a badge overlay on the Avatar with a tap-to-remove interaction.
5. WHEN a User taps the remove control on an Accessory badge, THE Accessory_Compositor SHALL remove that Accessory from the Accessory_Layer without modifying the base Outfit image.
6. IF the Wardrobe_Parser cannot identify a valid accessory from the submitted Accessory_Prompt, THEN THE Drape_App SHALL display an error message indicating the prompt could not be interpreted as an accessory.
7. FOR ALL valid Accessory_Prompts, parsing then formatting then parsing the prompt SHALL produce an equivalent Accessory definition (round-trip property).

### Requirement 7: Accessory Shelf Management

**User Story:** As a user, I want a shelf of my saved accessories so I can toggle them on/off with a single tap.

#### Acceptance Criteria

1. THE Drape_App SHALL display the Accessory_Shelf panel showing all of the User's saved Accessories.
2. WHEN a User taps an Accessory on the Accessory_Shelf, THE Accessory_Compositor SHALL toggle that Accessory on or off in the current Outfit's Accessory_Layer.
3. THE Drape_App SHALL visually indicate which Accessories on the Accessory_Shelf are currently active in the Outfit.
4. WHEN a User adds a new Accessory via Accessory_Prompt, THE Drape_App SHALL save the Accessory to the Accessory_Shelf for future reuse.

### Requirement 8: Accessory Suggestions

**User Story:** As a user, I want the app to suggest accessories that pair well with the current outfit so I can discover combinations I wouldn't have thought of.

#### Acceptance Criteria

1. WHEN an Outfit is generated or modified, THE Recommendation_Engine SHALL suggest Accessories that complement the current Outfit based on the Occasion_Context, Wardrobe color palette, and historical Accessory usage.
2. THE Drape_App SHALL display Accessory suggestions in a panel below the Accessory_Shelf.
3. WHEN a User selects a suggested Accessory, THE Accessory_Compositor SHALL add that Accessory to the current Outfit's Accessory_Layer.
4. THE Recommendation_Engine SHALL provide a brief explanation for each suggested Accessory describing why the Accessory complements the current Outfit.

### Requirement 9: Save Outfit with Personal Note

**User Story:** As a user, I want to save an outfit with an optional personal note so I can remember the moment.

#### Acceptance Criteria

1. THE Drape_App SHALL display a save button on the Outfit card.
2. WHEN a User taps the save button, THE Drape_App SHALL prompt the User to enter an optional Personal_Note of maximum 280 characters.
3. WHEN a User confirms the save action, THE Drape_App SHALL store a Saved_Outfit record containing: Outfit name, Occasion_Context, Wardrobe_Item identifiers, active Accessory identifiers, Personal_Note, and timestamp.
4. WHEN a Saved_Outfit is successfully stored, THE Drape_App SHALL display a confirmation toast notification.
5. WHEN a Saved_Outfit is successfully stored, THE Drape_App SHALL change the save button state to display "✓ Saved".
6. WHEN the User modifies the current Outfit after saving, THE Drape_App SHALL reset the save button to its default unsaved state.
7. IF a User submits a Personal_Note exceeding 280 characters, THEN THE Drape_App SHALL prevent submission and display a character limit warning.

### Requirement 10: Outfit Journey Timeline

**User Story:** As a user, I want to view all my saved outfits in a timeline grouped by month so I can see my style evolving.

#### Acceptance Criteria

1. WHEN a User navigates to the Outfit Journey screen, THE Drape_App SHALL display all Saved_Outfits in a chronological timeline grouped by month.
2. THE Drape_App SHALL display each Saved_Outfit card with: a mini Avatar thumbnail, Occasion_Context badge, Accessory emojis, date, and Personal_Note excerpt.
3. THE Drape_App SHALL provide filter chips (All, Work, Casual, Night Out) on the Outfit Journey screen.
4. WHEN a User selects a filter chip, THE Drape_App SHALL display only Saved_Outfits matching the selected Occasion_Context.
5. WHEN the "All" filter chip is selected, THE Drape_App SHALL display all Saved_Outfits regardless of Occasion_Context.
6. WHEN a User has zero Saved_Outfits, THE Drape_App SHALL display an empty state with a clear call-to-action directing the User to create and save an Outfit.
7. WHEN a User taps a Saved_Outfit card, THE Drape_App SHALL expand the card to show the full Outfit detail including full-size Avatar image, all Wardrobe_Items, all Accessories, and the complete Personal_Note.

### Requirement 11: Outfit Journey Filtering by Life Chapters

**User Story:** As a user, I want to look back at what I wore during different life chapters so I can understand how my style has changed.

#### Acceptance Criteria

1. THE Drape_App SHALL allow the User to browse the Outfit Journey timeline by scrolling through monthly groupings.
2. WHEN a User scrolls through the Outfit Journey, THE Drape_App SHALL display month and year headers to delineate time periods.
3. THE Drape_App SHALL maintain chronological ordering of Saved_Outfits within each monthly group, with the most recent Outfit appearing first.

### Requirement 12: Purchase-Linked Accessory Recommendations

**User Story:** As a user, I want the app to recommend accessories I don't own based on my existing wardrobe and style patterns so I can discover new pieces.

#### Acceptance Criteria

1. THE Recommendation_Engine SHALL suggest Accessories that the User does not currently own, based on the User's Wardrobe color palette, Style_Profile, and historical Accessory usage patterns.
2. THE Drape_App SHALL display each recommended Accessory with: item description, explanation of why the Accessory complements the User's style, and a link to purchase the item.
3. WHEN a User taps a purchase link on a recommended Accessory, THE Drape_App SHALL open the linked retailer page in a new browser tab.

### Requirement 13: Proactive Daily Outfit Suggestion

**User Story:** As a returning user, I want the app to study my outfit history and proactively suggest what to wear today so I don't have to start from scratch.

#### Acceptance Criteria

1. WHEN a returning User opens the Today's Look screen, THE Recommendation_Engine SHALL generate a proactive Outfit suggestion based on the User's Outfit history, Style_Profile, and current Occasion_Context.
2. THE Drape_App SHALL display the proactive Outfit suggestion on the Today's Look screen as the default view.
3. WHEN a User dismisses or modifies the proactive suggestion, THE Drape_App SHALL allow the User to generate a new Outfit using the standard generation flow.
